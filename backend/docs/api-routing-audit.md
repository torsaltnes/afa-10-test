# API Routing Audit

Audit of all 404 issues discovered and resolved during the OpenAPI / routing remediation sprint (2026-04-30).

| Endpoint / Path | HTTP Verb | Before-Fix Observed Result | Root Cause | Remediation | After-Fix Expected Status | Automated Test |
|---|---|---|---|---|---|---|
| `/openapi/v1.json` | GET | 404 in non-Development environments | `MapOpenApi()` was guarded inside `if (app.Environment.IsDevelopment())`, so the JSON document was never registered in staging / test / production | Removed environment guard; moved `MapOpenApi("/openapi/{documentName}.json")` into `MapOpenApiEndpoints()` which is always called | 200 OK | `OpenApiDocumentTests.GetOpenApiDocument_ReturnsOkAndJson` |
| `/api/docs` | GET | 404 (route not registered at all) | No browser documentation UI endpoint existed in any environment | Added `OpenApiEndpoints.MapOpenApiEndpoints()` that maps a lightweight `Results.Redirect` from `/api/docs` to the Scalar UI at `/scalar/v1`; `.ExcludeFromDescription()` keeps it off the schema | 302 ‚Üí 200 HTML | `ApiDocumentationUiTests.GetApiDocs_ReturnsRedirect_ToScalarUi`, `ApiDocumentationUiTests.GetApiDocs_DoesNotReturnNotFound` |
| `/api/health` | GET | 404 (route not registered) | Health endpoint was originally registered only at `/health`, outside the `/api/*` namespace used by all other endpoints; callers assuming the standard `/api/` prefix received 404 | Added canonical `/api/health` mapping; original `/health` retained as backward-compatible alias | 200 OK | `HealthEndpointsTests.GetHealth_CanonicalRoute_ReturnsOk`, `DocumentedEndpointStatusTests.GetHealth_CanonicalRoute_Returns200` |
| `/api/deviations` (GET, POST) | GET / POST | Ambiguous ‚Äî route was mapped as `"/"` within the group, producing path `/api/deviations/` (trailing slash); clients calling `/api/deviations` could receive 404 depending on runtime behaviour | `MapGet("/", ...)` and `MapPost("/", ...)` within the `/api/deviations` group creates a trailing-slash path that does not canonically match `/api/deviations` | Changed route pattern from `"/"` to `string.Empty` in `MapGet`/`MapPost` so the canonical path is definitively `/api/deviations` without trailing slash | 200 OK (GET) / 201 Created (POST) | `DeviationEndpointRouteTests.GetDeviations_CanonicalRoute_ReturnsOk`, `DeviationEndpointRouteTests.PostDeviations_CanonicalRoute_IsReachable` |
| `/api/deviations/export` | GET | Potential 404 if the route was incorrectly captured by `/{id:guid}` before export was registered | Export route must be registered before the `/{id:guid}` capture segment; if order was changed the literal segment `/export` would have matched the GUID constraint and returned 404 or a deserialization error | Verified registration order in `MapDeviationEndpoints`: `/export` is always registered before `/{id:guid}` | 200 OK with `text/csv` | `DeviationEndpointRouteTests.GetDeviationsExport_RouteIsReachable_Returns200`, `DocumentedEndpointStatusTests.GetDeviationsExport_Returns200WithCsvContentType` |
| OpenAPI schema missing metadata | N/A | OpenAPI operations had no `operationId`, `summary`, `description`, or response codes in the generated document | No `WithName`, `WithSummary`, `WithDescription`, `Produces`, or `Accepts` calls on any endpoint | Added complete metadata to every operation in `HealthEndpoints`, `DashboardEndpoints`, and `DeviationEndpoints`; removed deprecated `WithOpenApi()` call (ASPDEPR002 in .NET 10) | Schema reflects reality | `OpenApiDocumentTests.GetOpenApiDocument_ContainsExpectedOperationsAndResponses` |
| `/scalar/v1` (Scalar UI) | GET | 404 (no docs UI package installed) | `Microsoft.AspNetCore.OpenApi` does not include a browser UI; no helper package was referenced | Added `Scalar.AspNetCore` 2.14.8; mapped via `app.MapScalarApiReference(options => options.WithOpenApiRoutePattern("/openapi/{documentName}.json"))` | 200 OK HTML | `ApiDocumentationUiTests.GetScalarUi_ReturnsSuccess` |
